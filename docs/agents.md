# Agents — The Last-Minute Life Saver

## AI Agent Architecture

This document describes how Gemini AI acts as an autonomous agent within the application, and the multi-agent patterns used for different tasks.

---

## Agent Overview

The Last-Minute Life Saver employs two specialized AI agents, both powered by Gemini 1.5 Flash, operating with distinct roles and personas:

---

## Agent 1: The Decomposer Agent

**Role**: Emergency Planning Specialist  
**Model**: Gemini 1.5 Flash  
**Mode**: Batch (single call, structured JSON output)  
**Temperature**: 0.4 (reliability-focused)

### Persona
```
"You are an elite AI Emergency Management System called 'The Last-Minute Life Saver.'
Your mission: Transform chaotic deadline panic into an immediately executable action plan."
```

### Input
Raw, unstructured panic text from the user:
> "I have an engineering paper due at 8 AM tomorrow and I haven't started. I still need an abstract, 3 body sections, and a bibliography."

### Processing
1. **Intent extraction**: Parse the deadline, task type, and missing components
2. **Time calculation**: Anchored to injected `currentISOTime` — calculates exact hours remaining
3. **Urgency scoring**: Maps time delta to score 1–10
4. **Step decomposition**: Breaks task into 4–12 micro-steps that fit within available time
5. **Prompt generation**: Creates specific, immediately-usable `ai_starter_prompt` for each step

### Output (Structured JSON)
```json
{
  "task_name": "IEEE Paper: ML in Smart Grid Optimization",
  "true_deadline": "2024-01-15T08:00:00.000Z",
  "urgency_score": 9,
  "action_steps": [...]
}
```

### Agent Constraints
- **Schema enforcement**: `responseMimeType: "application/json"` prevents free-text responses
- **Time realism**: Total step durations cannot exceed available time
- **Specificity requirement**: Each `ai_starter_prompt` must be immediately usable
- **Safety guardrails**: Content safety filters + Vertex AI schema validation

## Agent 2: The Executor Agent

**Role**: Expert Task Execution Assistant  
**Model**: Gemini 1.5 Flash  
**Mode**: Streaming (real-time SSE output)  
**Temperature**: 0.7 (creativity-focused)

### Persona
```
"You are an expert execution assistant. The user is under extreme time pressure.
Provide an immediately usable, high-quality response.
Format your response in clean markdown with clear sections.
Be direct, actionable, and thorough. Start immediately — no preamble."
```

### Input
The `ai_starter_prompt` generated by the Decomposer Agent, enriched with task context:
```
TASK CONTEXT: IEEE Engineering Paper Due 8 AM — Task ID: task_123
CURRENT STEP: Write abstract (write)

Write a 300-word abstract for an engineering research paper on 
"Machine Learning in Smart Grid Optimization"...
```

### Processing
- Streams markdown content in real-time via `generateContentStream()`
- Rendered live in the `ExecutionWorkspace` component using the `marked` library

### Output
Streamed markdown content tailored to the specific step.

## Agent Orchestration Pattern

```
User Input → [Decomposer Agent] → Structured Plan
                                        │
                           ┌────────────┴───────────────┐
                           │    For each action step:   │
                           │                            │
                    User clicks "Execute"               │
                           │                            │
                    [Executor Agent] ◄──────────────────┘
                           │
                    Streamed Output → Execution Workspace
```

The two agents work in sequence — the Decomposer creates a structured plan, and the Executor handles each step on-demand as the user executes them.

## Autonomous Intervention Agent

**Role**: Proactive Deadline Monitor  
**Runtime**: Cloud Function (Node.js 20)  
**Trigger**: Cloud Scheduler (every 15 minutes)  
**Mode**: Rules-based + FCM push

This is not an LLM agent — it is an autonomous monitoring agent that:
1. **Queries Firestore** for tasks approaching deadline (< 120 minutes)
2. **Evaluates urgency** using the `recalculateUrgencyScore()` function
3. **Selects intervention target** — first incomplete step
4. **Composes notification** with urgency-appropriate messaging
5. **Delivers via FCM** to the user's registered device

### Intervention Decision Logic
```javascript
if (minutesRemaining <= 120 && task.archived === false) {
  const incompleteSteps = steps.filter(s => !s.completed);
  if (incompleteSteps.length > 0) {
    sendInterventionNotification({
      stepTitle: incompleteSteps[0].title,
      minutesRemaining,
      urgencyScore: recalculateUrgencyScore(task.true_deadline)
    });
  }
}
```

## Future Agent Extensions

The architecture supports adding additional specialized agents:

| Proposed Agent | Role | Trigger |
|----------------|------|---------|
| **Resource Fetcher Agent** | Finds and summarizes relevant research papers | On "research" step execution |
| **Progress Analyzer Agent** | Adjusts remaining steps based on completion pace | Every 30 minutes |
| **Submission Validator Agent** | Checks deliverable against requirements | On "submit" step |
| **Recovery Agent** | Generates emergency plan when deadline passes | On deadline expiry |
